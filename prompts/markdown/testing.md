# Testing Scenarios

## Scenario 1: Simple Case for Testing

```
Use this to test basic functionality:

CHARGES: Trespassing

FACTS:
- Defendant allegedly entered property marked "No Trespassing"
- Property owner called police
- Defendant claims was lost and seeking help
- No damage to property occurred

EVIDENCE:
- Property owner testimony
- "No Trespassing" signs (photos)
- Defendant's statement to police

This tests:
- Basic opening/closing structure
- Simple evidence presentation
- Credibility assessment
- Straightforward legal application
```

## Scenario 2: Constitutional Issue for Testing

```
Use this to test RAG integration:

CHARGES: Possession of Stolen Property

FACTS:
- Police stopped defendant for traffic violation
- During stop, saw suspicious item in plain view
- Asked for consent to search vehicle
- Defendant said "I guess so"
- Found stolen laptop in trunk

EVIDENCE:
- Stolen laptop
- Police testimony about stop and search
- Defendant's ambiguous consent

This tests:
- Fourth Amendment issues requiring case law
- RAG search for "consent search voluntary" cases
- Suppression motion procedure
- Conditional proceeding (what if evidence suppressed?)
```

## Scenario 3: Complex Multi-Issue Case

```
Use this to test full system:

CHARGES: Aggravated Assault

FACTS:
- Bar fight resulting in serious injury
- Multiple witnesses with conflicting accounts
- Video shows altercation but not who started it
- Defendant claims self-defense
- Victim required hospitalization

EVIDENCE:
- Victim testimony
- Three bar patron witnesses (varying accounts)
- Bartender testimony
- Video footage
- Medical records
- Defendant's testimony (if testifies)

This tests:
- Witness credibility assessment
- Self-defense jury instruction
- Evidence conflicts
- Burden of proof on self-defense claim
- Full trial procedure from opening to closing
```
